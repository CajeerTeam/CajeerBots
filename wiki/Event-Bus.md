# Шина событий

Шина событий связывает адаптеры, модули и плагины через единый контракт событий.

## Runtime-слой

В базовом каркасе используется локальная in-memory шина. Она нужна для разработки, API-диагностики и проверки маршрутизации. Боевой транспорт может быть заменён внешним слоем: PostgreSQL, Redis Streams, Kafka или другим брокером.

## Маршрутизация

События проходят через `EventRouter`:

```text
adapter.started     системная обработка адаптера
command.received    реестр команд
message.*           обработчики модулей
plugin.*            обработчики плагинов
```

Если обработчик не назначен, событие не теряется: результат маршрутизации содержит причину, почему событие не обработано.
